100 /*
101  * kiss_fft_alloc
102  *
103  * Initialize a FFT (or IFFT) algorithm's cfg/state buffer.
104  *
105  * typical usage: kiss_fft_cfg mycfg=kiss_fft_alloc(1024,0,NULL,NULL);
106  *
107  * The return value from fft_alloc is a cfg buffer used internally
108  * by the fft routine or NULL.
109  *
110  * If lenmem is NULL, then kiss_fft_alloc will allocate a cfg buffer using malloc.
111  * The returned value should be free()d when done to avoid memory leaks.
112  *
113  * The state can be placed in a user supplied buffer 'mem':
114  * If lenmem is not NULL and mem is not NULL and *lenmem is large enough,
115  * then the function places the cfg in mem and the size used in *lenmem
116  * and returns mem.
117  *
118  * If lenmem is not NULL and ( mem is NULL or *lenmem is not large enough),
119  * then the function returns NULL and places the minimum cfg
120  * buffer size in *lenmem.
121  * */
